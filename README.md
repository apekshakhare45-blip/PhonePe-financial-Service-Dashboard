## Digital Payments Analysis Dashboard
## Project Overview
This project analyzes digital payment transaction data using Power BI to understand user behavior, transaction trends, service performance, and failure patterns.
The dashboard provides interactive insights that help stakeholders identify growth opportunities, monitor success rates, and improve digital payment services.

## Objectives
- Analyze transaction volume and revenue trends
- Identify most used payment services
- Monitor transaction success vs failure rates
- Understand monthly growth patterns
- Provide actionable insights for business decision-making

## Tools & Technologies
- Power BI
- Power Query (Data Cleaning & Transformation)
- DAX (Measures & KPIs)

## Dataset Description
The dataset includes:
- Transaction details
- Service category (Insurance, Loans, etc.)
- Payment amount
- Transaction status (Success / Failed)
- Failure reason
- Date information
- User activity summary

## Dashboard Features
- KPI cards (Total Transactions, Success Rate, Revenue)
- Service-wise transaction distribution
- Monthly trend analysis
- Failure pattern visualization
- Interactive filters and slicers for deeper analysis

## Key Insights
- Certain services dominate overall transaction volume
- Noticeable spikes in transactions during specific months
- Failure rates vary across services indicating potential system issues
- Revenue contribution is uneven across payment categories

## Business Recommendations
- Improve infrastructure for high-failure services
- Promote underperforming services through targeted campaigns
- Focus on peak transaction periods for scaling systems
- Optimize user experience to improve success rate
